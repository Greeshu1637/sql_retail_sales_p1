# 📊 Retail Sales Performance Analysis using SQL

## 📌 Project Overview

This project presents an end-to-end Retail Sales Analysis using SQL, focusing on transforming raw transactional data into meaningful business insights. The objective is to analyze customer purchasing behavior, sales trends, and product performance to support data-driven business decisions.

The project follows a structured analytics workflow that includes database creation, data cleaning, exploratory data analysis (EDA), and business-focused SQL queries. Through this analysis, various key performance indicators (KPIs) are generated to evaluate overall business performance and identify opportunities for growth.

---

## 🎯 Business Problem

Retail businesses generate large volumes of transactional data every day. Without proper analysis, valuable information about customer behavior, product demand, and sales performance remains hidden.

The goal of this project is to use SQL to answer real-world business questions such as:

* Which product categories generate the highest revenue?
* Which customer segments contribute the most to sales?
* How do sales vary across different months?
* What are the purchasing patterns of customers?
* Which transactions represent high-value sales?

By answering these questions, businesses can make informed decisions regarding marketing strategies, inventory planning, and customer engagement.

---

## 🎯 Project Objectives

* Design and create a structured retail sales database.
* Perform data cleaning to improve data quality.
* Conduct exploratory data analysis using SQL.
* Analyze customer purchasing behavior.
* Evaluate product category performance.
* Identify monthly and yearly sales trends.
* Generate business insights through SQL queries.
* Present actionable recommendations based on the analysis.

## 📂 Dataset Description

The dataset contains retail transaction records, including customer demographics, product categories, purchase quantities, pricing information, and total sales.

### Dataset Features

* Transaction ID
* Sale Date
* Sale Time
* Customer ID
* Gender
* Age
* Product Category
* Quantity Purchased
* Price per Unit
* Cost of Goods Sold (COGS)
* Total Sale Amount

This dataset provides sufficient information to perform comprehensive sales analysis and derive meaningful business insights.
